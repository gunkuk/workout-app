# UI v2 — Boostcamp 클론 재설계 (스크린샷 리서치 기반)

> 근거: Boostcamp 공식 App Store 스크린샷 픽셀 샘플링 + Liftosaur/wger/Iron 오픈소스 패턴 + 헬스장 UX 리서치(2026-07-10 워크플로).
> 하드 게이트: **기존 258+ 테스트 무수정 통과** — data-testid·aria-label·핸들러 시맨틱·기존 렌더 텍스트("67.5kg × 8" 형태, "완료", "워밍업" 등 테스트가 찾는 문자열)는 전부 보존. DOM 구조·CSS·시각 배치는 자유.

## 디자인 토큰 (Boostcamp 실측값)
```css
:root {
  --bg: #1a1a1a;            /* 캔버스(#000과 #262626 사이) */
  --surface: #262626;        /* 카드/시트 — Boostcamp 실측 */
  --surface2: #353535;       /* 입력 박스/raised — 실측 */
  --surface3: #3c3c3c;
  --border: #3a3a3a;
  --text: #ffffff;
  --muted: #999999;          /* 실측 secondary */
  --gold: #fcc100;           /* CTA·완료 체크·운동명 — 실측 */
  --orange: #fe7100;         /* 분석 큰 숫자·차트 라인 — 실측 */
  --teal: #25d5b8;           /* 보조 차트 — 실측 */
  --badge-warmup: #7d6417;   /* W 배지 — 실측 */
  --badge-amrap: #623232;    /* F 배지 — 실측 */
  --success: #fcc100;        /* 완료 = 골드 (Boostcamp 방식) */
  --danger: #ff5c5c;
}
```
폰트: 기존 시스템 스택 유지. 숫자는 tabular-nums + bold.

## 오늘 화면 (전면 재구성 — 핵심)

**스티키 헤더** (스크롤해도 고정, bg var(--bg), 하단 보더):
- 좌: day명(예: "벤치 volume + OHP") 16px bold — 길면 말줄임
- 우: ⚙ 버튼(44px, 배너와 절대 안 겹치게 헤더 안으로 이동)
- 세션 완료 버튼: **전 세트 완료 시에만**(기존 로직 그대로) 헤더 우측에 골드 pill로 등장 — "세션 완료" 텍스트 유지. 미완료 시 대신 진행 카운트("12/20") muted 표시.

**운동 섹션** (Boostcamp 스타일, 운동당 1블록):
- 헤더 행: **운동명 한글 골드 볼드 17px**(exerciseLibrary의 name — 지금 없던 걸 추가: PlannedSlot.exerciseId → exerciseInfo(id)?.name ?? id) + 우측에 "N/M" 세트 진행 카운트(muted).
- 그 아래 한 줄: 라벨(T1/T2/accessory)을 11px muted uppercase + 스킵/통증일 버튼을 **작은 텍스트 버튼**으로 우측 정렬(현재 거대 카드 버튼 → 컴팩트하게. 텍스트·핸들러 유지).
- **세트 테이블**: 4컬럼 그리드 `[배지 40px | 목표(1fr) | 조정(auto) | 체크 44px]`
  - 컬럼 헤더는 생략(Boostcamp는 있지만 우리 데이터가 단순 — 과밀 방지).
  - **배지**: 워밍업 행 = "W" 배지(--badge-warmup bg, 흰 12px bold, 라운드 6px) / AMRAP(amrapRole 있는 세트) = "F" 배지(--badge-amrap) / 일반 = 세트번호 muted.
  - **목표 셀**: "67.5kg × 8" (15px, 완료 전 흰색·완료 후 muted) — 기존 텍스트 포맷 그대로(테스트 보존). 아래 서브라인 11px muted로 원판("25 + 2.5") — PlateBreakdown을 이 위치로 이동.
  - **조정 셀**: 기존 스테퍼 4버튼(aria-label 보존 필수)을 **완료 전에는 숨기고(display:none 아닌 조건부 렌더는 시맨틱 변경이므로: CSS로 축소 — 32px 소형 2×2 그리드 or 한 줄 소형)**… ⚠️ 단순화: 스테퍼를 28px 소형 버튼으로 줄여 목표 셀과 체크 사이에 한 줄로 유지(아이콘만: − + 는 무게, −렙 +렙은 그대로 텍스트 유지해야 하면 폭 문제 → "−렙"/"+렙" 텍스트는 aria-label이 아니라 버튼 텍스트라면 변경 불가. 확인 후: 버튼 내부 텍스트가 테스트에서 사용되면 유지하되 11px로 축소+세로 2단(무게단/렙단)으로 폭 절약).
  - **체크 원**(행의 완료 상태): 우측 40px 원 — 미완: 2px 보더 var(--muted) 원 + 희미한 ✓ / 완료: **골드 채움 + 검정 ✓** (Boostcamp 시그니처). 행 전체 탭=완료(기존), 완료 후 재탭=수정(기존) 전부 유지. 기존 "완료" 텍스트 pill이 테스트에서 쓰이면 체크 원 안/옆에 sr-only 또는 소형으로 보존.
  - 행 높이 52~56px, 행 간 구분은 보더(카드 개별 분리 X — 운동 블록이 하나의 surface 카드, 행은 divider).
- **자유입력(악세사리 needsInit)**: 같은 테이블 형태에서 kg/렙이 **다크 입력 박스**(--surface2, 볼드 흰 숫자, 60px 폭 — Boostcamp의 Lbs/Reps 박스 스타일) + 체크 원이 제출 버튼 역할("완료" 버튼 텍스트 보존).
- **워밍업 행**: 일반 행과 같은 테이블에 W 배지로 편입(현재 별도 리스트 → 통합). "워밍업 20kg × 10" 텍스트는 테스트가 찾을 수 있으니 행 내 텍스트로 보존.
- **휴식 타이머**: 운동 블록 하단이 아니라 **화면 하단 고정 플로팅 pill**(Liftosaur 패턴 — 내비바 위 12px, 우측, --surface3 bg, 골드 숫자, 28px bold). 기존 RestTimer 컴포넌트/시맨틱 유지하고 위치·스타일만.

**ProposalCard**: 골드 좌측 보더 + --surface, 옵션 버튼 골드 pill(검정 텍스트 — Boostcamp CTA 방식), 동결/보류 ghost.

## 히스토리/설정/온보딩
- 히스토리: 세션 행 카드(surface), 날짜 포맷 "7월 10일 (금)" 같은 한국어 짧은 포맷으로 표시만 변경(ISO 원문은 title 속성으로 보존 — 테스트가 ISO 텍스트를 찾으면 유지 필요: 확인 후 테스트가 찾는 문자열은 보존).
- 설정: 섹션 카드화 유지, 저장/전환 버튼 골드.
- 온보딩: 입력 박스 --surface2, "시작하기" 골드 전폭.
- 설치 배너: 캔버스 최상단 유지하되 ⚙와 분리됐으므로 겹침 해소. 닫기 버튼 우측 여백 확보.
