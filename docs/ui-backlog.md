# UI 백로그 — VLM 검수 라운드 2 관찰 (2026-07-10, 다음 라운드 후보)

> 컨트롤러가 Playwright 스크린샷(375×812)을 직접 보고 남긴 크리틱. 실기기 사용자 피드백과 합쳐 라운드 3에서 처리.

## 관찰 (심각도순)
1. **히스토리 빈 상태가 밋밋** — "아직 기록된 세션이 없습니다"만. CTA 추가 제안: "오늘 탭에서 첫 세션을 시작하세요" + 오늘 탭 이동 링크.
2. **설치 배너가 2줄로 김**(375px에서) — 문구 축약("홈 화면에 추가로 설치하세요") 또는 아이콘화 검토.
3. **설정 정보 구조** — "URL에서 가져오기"가 모드 설정 사이에 끼어 있음. 가져오기(파일+URL)끼리 묶고 모드 설정을 분리하는 재배치 검토(마크업 이동 — 테스트 영향 확인 필요).
4. **스테퍼(−/+/−렙/+렙) 존재감** — 미완료 세트에 상시 노출 중. Boostcamp처럼 탭한 세트(현재 세트)에만 노출하는 안 검토 — 단 테스트가 aria-label 존재를 전제하므로 조건부 렌더 시 테스트 조정 필요(계약 변경 — 신중히).
5. **TM 수동 편집 정렬** — 라벨 폭이 제각각이라 입력박스가 들쭉날쭉. `li`를 2컬럼 그리드(라벨 고정폭)로.
6. **분석 빈 상태** — 외부 세션 추가 UI가 데이터 없을 때도 노출. 세션 생기기 전엔 숨기거나 안내문 아래로.
